# ADR-0001: Размерность embedding-вектора и провайдер

- **Дата**: 2026-04-25
- **Статус**: Accepted
- **Решение принял**: Egor (владелец)
- **Связанная точка решения**: D1 (из `ROADMAP.md` раздел 2)
- **Связанные ADR**: —

## Контекст

Pipeline содержит стадию `EmbeddingStage` (4-я в каскаде из 8) — превращает текст
ментиона в вектор фиксированной размерности `N`. Дальше `SemanticFilter` использует
этот вектор для cosine-сравнения с предрасчитанными векторами тем проекта,
отсекая ~95% шума до дорогой LLM-классификации. Вектор хранится в Postgres в
колонке `vector(N)` через расширение pgvector.

**Точка невозврата на уровне SQL**: размерность `N` фиксируется при создании
таблицы. Смена `N` потом = пересобрать индекс HNSW + переэмбеддить весь корпус
ментионов в БД + миграция схемы. Смена **провайдера** при той же `N` — дешёвая:
переэмбеддить корпус новым API, схема не меняется. Поэтому реальная точка
невозврата — размерность.

## Опции

| # | Провайдер | Dim native | Цена за 1M tok | Multilingual | Hosting |
|---|-----------|------------|----------------|--------------|---------|
| 1 | Voyage 3.5 | 1024 | $0.06 | да (вкл. русский) | API |
| 2 | OpenAI text-embedding-3-small | 1536 (можно 1024 через `dimensions`) | $0.02 | да | API |
| 3 | OpenAI text-embedding-3-large | 3072 (можно 1024) | $0.13 | да | API |
| 4 | BGE-M3 self-hosted | 1024 | $0 за токен, $20–80/мес GPU/CPU | да | свой сервер |

Выбор размерности отдельно: 1024 — единственная размерность, нативно
поддерживаемая всеми четырьмя опциями. 1536 запирает на OpenAI. 3072 — overkill
для Phase 0 (большой индекс, медленный HNSW). 768 — устаревшие модели.

## Решение

**Размерность = 1024. Провайдер = Voyage 3.5 (`voyage-3.5`).**

Обоснование:
- 1024 — провайдер-портабельная размерность; через `IEmbedder` Protocol можно
  мигрировать на любого из четырёх без миграции SQL-схемы. Это страховка от
  риска маленького провайдера.
- Voyage 3.5 — лучшее качество retrieval-задач на MTEB при сопоставимой цене.
  Anthropic официально рекомендует Voyage как embedding-партнёра для пользователей
  Claude (`docs.anthropic.com`).
- Не BGE self-hosted — операционная стоимость ($20–80/мес GPU + поддержка)
  выше экономии на токенах ($0.36/мес у Voyage при оценке 6M токенов/мес).
- Не OpenAI 3-small — экономия на embedding на ~3× меньше, чем потеря на
  LLM-стадии из-за худшего ретривал-качества (больше шума прорывается в Haiku).

## Последствия

**Становится возможным:**
- SQL-схема `mentions_embeddings.vector vector(1024)` фиксируется в первой
  миграции.
- Контракт `IEmbedder` декларирует `dimensions: int = 1024` как часть
  публичного интерфейса.
- Migration на OpenAI 3-small/3-large или BGE-M3 — за один батч пересчёта,
  без изменения схемы.

**Становится дорогим / невозможным:**
- Смена размерности (например, на 2048 для voyage-3-large) — миграция всего
  корпуса + пересборка HNSW + изменение схемы. Эквивалентно «переезду».
- Использование моделей с native dim ≠ 1024 без поддержки усечения (например,
  старые BGE-base 768) — потребуется проксирующий слой или смена размерности.

**Что делать на следующих шагах:**
- В `core/CLAUDE.md` (создаётся в E0) — добавить контракт `IEmbedder` с полями
  `dimensions: int`, `model_id: str`, `cost_per_1m_tokens: Decimal`.
- В `storage/schema.sql` (E1, ветка 1) — `vector(1024)` в таблице
  `mentions_embeddings`.
- В `.env.example` — `VOYAGE_API_KEY=...`.
- В `processing/stages/embedding.py` (E2a) — реализация `VoyageEmbedder` через
  HTTP-клиент (нет официального async-SDK на момент решения, но есть REST).

**Риски и митигации:**
- *Voyage поднимает цены или уходит с рынка*: миграция на OpenAI 3-small
  (`dimensions=1024`) или BGE-M3 (нативно 1024) — за один батч пересчёта. Стоимость
  миграции при корпусе 100к ментионов × 200 токенов = 20M токенов: на OpenAI
  3-small ~$0.40 единоразово.
- *Качество voyage-3.5 окажется хуже ожидаемого на наших данных (русский /
  технический сленг)*: можно протестировать voyage-3-large (тот же `dim=1024`,
  но другая модель) или BGE-M3 без миграции схемы.
